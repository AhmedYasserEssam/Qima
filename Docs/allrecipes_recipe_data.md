# Allrecipes Recipe Data

This document describes the normalized Allrecipes recipe dataset generated by `scrappers/scrape_allrecipes_food.py`.

The generated files live under `data/Recipes/`. The `data/` directory is intentionally ignored by Git because these files can be large and are reproducible local artifacts.

## Current Local Snapshot

The inspected local JSONL snapshot is `data/Recipes/allrecipes_recipes.jsonl`.

| Property | Value |
| --- | --- |
| Source | Allrecipes |
| Format | JSON Lines, one recipe object per line |
| Local record count | 15,514 recipes |
| Default generator output | `data/Recipes/allrecipes_recipes.jsonl` |
| CSV export | `data/Recipes/allrecipes_recipes.csv` |
| Primary key candidate | `source` + `recipe_id` |

## File Layout

| File | Purpose |
| --- | --- |
| `allrecipes_recipes.jsonl` | Full normalized recipe objects with nested ingredients, directions, nutrition, dietary flags, allergens, and quality metadata. |
| `allrecipes_recipes.csv` | Flattened tabular export for quick inspection or loading into spreadsheet tools. JSON-like nested values are stored in `*_json` columns. |
| `allrecipes_recipes_100.*` | Small sample exports for development checks. |
| `allrecipes_enrichment_check.*` | Small validation/enrichment check outputs. |
| `13k-recipes.csv` | Older/raw recipe source file retained locally for reference. |

## Top-Level JSONL Schema

Each line in the JSONL file is a JSON object with these top-level fields.

| Field | Type | Description |
| --- | --- | --- |
| `source` | string | Source identifier, currently `allrecipes`. |
| `source_url` | string | Canonical recipe URL. |
| `recipe_id` | string | Source recipe ID extracted from the URL. |
| `stable_slug` | string | Stable URL slug used for readable identity. |
| `title` | string | Recipe title. |
| `cuisine` | string or null | Inferred or source-derived cuisine label. |
| `category` | string or null | Recipe category such as `Dinner`, `Side Dish`, or `Dessert`. |
| `meal_type` | string or null | Normalized meal type when available. |
| `author_name` | string or null | Source author name. |
| `servings` | number or null | Number of servings. |
| `times` | object | Prep, cook, and total time in minutes. |
| `ingredients` | array | Structured ingredient objects. |
| `directions_json` | array | Structured cooking steps. |
| `cooking_methods` | array | Normalized method tags such as `baking`, `simmering`, or `smoking`. |
| `equipment` | array | Equipment detected from directions. |
| `nutrition_facts_raw` | object or null | Raw source nutrition payload. |
| `nutrition_per_serving` | object | Numeric per-serving nutrition fields. |
| `nutrition_quality` | object | Nutrition provenance and known quality issues. |
| `tags` | array | Source and inferred tags. |
| `dietary_flags` | object | Boolean or `unknown` dietary classifiers. |
| `dietary_confidence` | object | Confidence levels for selected dietary classifiers. |
| `allergen_flags` | array | Allergens directly detected from ingredient rules. |
| `possible_allergen_flags` | array | Allergens that may be present in packaged or ambiguous ingredients. |
| `allergen_basis` | object | Mapping of allergen to ingredient text that triggered it. |
| `allergen_confidence` | object | Confidence labels for allergen detections. |
| `packaged_ingredient_warnings` | array | Ingredients that should be checked against product labels. |
| `difficulty` | string or null | Estimated difficulty such as `easy`, `medium`, or `hard`. |
| `rating` | number or null | Source rating. |
| `review_count` | integer or null | Source review count. |
| `date_published` | string or null | Source publication timestamp. |
| `date_modified` | string or null | Source modification timestamp. |
| `attribution` | object | Attribution fields for source name and URL. |
| `data_quality_flags` | array | Dataset-level quality and provenance flags. |
| `normalization_warnings` | array | Normalizer warnings for the recipe. |
| `completeness_score` | number | Completeness score from `0.0` to `1.0`. |
| `normalization_quality_score` | number | Normalization quality score from `0.0` to `1.0`. |
| `recipe_quality_score` | number | Recipe quality score from `0.0` to `1.0`. |

## Quality Notes

Nutrition values are source-reported and should be treated as unverified unless `nutrition_quality.verified_by_qima` is true.

Dietary and allergen flags are inferred from ingredient text and rule-based normalization. Packaged or ambiguous ingredients may include `requires_label_check`, `possible_allergen_flags`, or `packaged_ingredient_warnings`; these should be surfaced to users instead of treated as definitive label claims.

Some source text may contain encoding artifacts from upstream pages. Recipes affected by parser or normalization issues should carry entries in `normalization_warnings` or `data_quality_flags`.

## Regeneration

Run the scraper from the repository root:

```powershell
py .\scrappers\scrape_allrecipes_food.py --output data\Recipes\allrecipes_recipes.jsonl --log-level INFO
```

For a small smoke test:

```powershell
py .\scrappers\scrape_allrecipes_food.py --max-urls 3 --output data\Recipes\allrecipes_sample.jsonl --log-level INFO
```
